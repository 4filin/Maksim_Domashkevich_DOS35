## Работа с GitHUB


1. Создаем репозиторий в GitHub и клонируем его на локальную машину
    git clone
   
   ![step1](/Lesson_2_Git/temp_l2/step1.png)

2.  Переносим папку с подготовленным отчетом Lesson_3_VM_OS в локальный репозиторий 
   С помощью команды git add . добавляем файлы/папки в индекс изменений  
    ![step2](/Lesson_2_Git/temp_l2/step2.png)

3. Создаем коммит и добавляем описание  
    ![step3](/Lesson_2_Git/temp_l2/step3.png)

4. Отправляем локальную ветку в репозиторий GitHUB
   git push

   ![step4](/Lesson_2_Git/temp_l2/step4.png)     

5. После проверки в репозиотории GIT, выявили неточность в описании путей к сриншотам. 
   Делаем изменения в локальном md файле создаем новый commit and push для исправления
   
   ![step5](/Lesson_2_Git/temp_l2/step5.png)

6. Готовим report под текщий урок GIT содержит только папку с Lesson 3 
    
   ![step6](/Lesson_2_Git/temp_l2/step6.png)

    создаем папку Lesson_1 и Lesson_2 добавляем туда report.md папку temp_l2 и файл gitcode.py для работы с кодом

   ![step7](/Lesson_2_Git/temp_l2/step7.png)

    Через консоль вводим команды (add, commit, push) и проверяем log 

    ![step8](/Lesson_2_Git/temp_l2/step8.png)
